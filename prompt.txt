# Grand Egyptian Museum Tourist Guide

## Overview

Grand Egyptian Museum Tourist Guide is an AI-powered museum assistant platform designed to enhance the tourist experience through computer vision, Retrieval-Augmented Generation (RAG), multilingual content management, and scalable backend architecture.

The system combines artifact image recognition, AI-generated historical explanations, multilingual monument information, and personalized user features into a unified platform connected through RESTful APIs and microservices.

The project follows a microservices-based architecture with separate services for:

* Backend API and business logic
* AI artifact recognition
* AI historical question answering and monument descriptions

---

# System Architecture

```text
Frontend / Mobile App
        ↓
Node.js Backend API
   ↙                  ↘
AI Recognition      RAG AI Guide
(FastAPI)           (Qwen + ChromaDB)
        ↓                  ↓
   PostgreSQL         Vector Database
```

---

# Core Features

## AI Artifact Recognition

Users can upload images of Egyptian artifacts and monuments. The backend forwards images to a FastAPI AI service powered by TensorFlow models to identify the artifact.

The system returns:

* Predicted artifact class
* Confidence score
* Matched monument data
* AI-generated historical description

---

## AI Historical Guide (RAG)

Integrated a Retrieval-Augmented Generation system using:

* Qwen 2.5 LLM
* ChromaDB vector database
* Semantic search embeddings

The AI guide can:

* Answer historical questions
* Generate monument descriptions
* Explain Egyptian history
* Provide contextual information for recognized artifacts

---

# AI Integration Workflow

## Artifact Recognition Flow

```text
User uploads image
        ↓
Backend receives multipart request
        ↓
Backend forwards image to ai-service
        ↓
AI predicts artifact class
        ↓
Backend maps AI class to monument
        ↓
Backend stores scan session
        ↓
Backend optionally requests RAG description
        ↓
Frontend receives full response
```

---

# RAG Workflow

```text
User asks question
        ↓
Backend sends request to rag-service
        ↓
ChromaDB retrieves relevant chunks
        ↓
Qwen 2.5 generates contextual answer
        ↓
Answer returned to backend
        ↓
Frontend displays AI response
```

# Running the Project

## 1. Start PostgreSQL

Ensure PostgreSQL is running locally on port 5432.

---

## 2. Run AI Recognition Service

```bash
cd ai-service
.\.venv\Scripts\Activate.ps1
uvicorn api.main:app --reload --port 8000
```

---

## 3. Run RAG Service

```bash
cd rag-service
.\.venv\Scripts\Activate.ps1
uvicorn main:app --reload --port 8001
```

---

## 4. Run Backend

```bash
cd backend
npm install
npm run seed
npm run dev
```
